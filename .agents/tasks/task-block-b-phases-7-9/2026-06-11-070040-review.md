# Search, performance instrumentation, and crash protection for EDMS (v2)

Block B adds three infrastructure layers to the EDMS frontend: a PostgreSQL full-text search query descriptor model (Phase 7), client-side performance instrumentation (Phase 8), and a resilience stack covering file validation, retry logic, circuit breaking, and structured error codes (Phase 9). All source files are additive with no modifications to pre-existing business logic. The v1 review raised 7 issues; 4 were directly fixed, 3 were deferred with justification.

**Watch for:** The sanitizer test suite doesn't exercise the newly added tsquery operator stripping (`&|!()`, `--`, `/*`) — the implementation is correct but a targeted test would prevent regressions if the regex is later modified (possible). The global inflight map remains a deferred architectural concern with no auto-expiry for hung promises (confirmed, acknowledged).

**Verdict**: APPROVED

## High-level view

The v1 concern about zero-delay backoff is resolved: `calculateBackoffDelay` now enforces a `MIN_DELAY_FLOOR = 100` ms, and the test suite explicitly asserts the floor when `Math.random()` returns 0.

The search term sanitizer now strips tsquery operators (`&`, `|`, `!`, `(`, `)`) and SQL comment sequences (`--`, `/*`, `*/`). Combined with the descriptor pattern (backend parameterizes), the injection surface is adequately reduced.

Performance mark accumulation is addressed by calling `clearPerformanceEntries(componentName)` whenever `durationsRef` exceeds `MAX_RENDER_HISTORY`. This periodic cleanup prevents timeline flooding during long-lived component lifetimes.

The four previously untested files (`imageLazyLoading`, `bundleSizeBudget`, `searchBenchmarkConfig`, `searchFiltersIntegration`) now have test files totaling 796 lines with comprehensive coverage of their public APIs.

The three deferred items (global inflight map lifecycle, localStorage saturation strategy, circuit breaker binary status) are reasonable deferrals: the first requires architectural scoping beyond this PR, the second is protected by the 50-entry cap, and the third is a deliberate design simplification.

1. **Sanitizer test coverage gap for tsquery operators** — The `sanitizeSearchTerm` tests verify quote/semicolon/backslash stripping and `--` removal (via the SQL injection test case), but no dedicated test asserts that `&`, `|`, `!`, `(`, `)`, or `/*` are stripped. If the regex is later modified, the tsquery stripping could regress without a test catching it. Add a test case like `sanitizeSearchTerm("valve & (pipe | tube)")` → `"valve  pipe  tube"`.
2. **Global inflight map hung-promise accumulation** — Acknowledged as deferred. The module-level `globalInflightMap` deletes keys only on promise settlement. A permanently-hung fetch leaves the key in the map forever. Acceptable for now given low concurrency, but should be addressed before scaling to high-throughput scenarios.

## Sanitizer regression risk

The `sanitizeSearchTerm` function now has four `replace` calls covering the original dangerous characters, tsquery operators, double-dash, and block-comment sequences. The test suite exercises the original character set (`'`, `"`, `` ` ``, `;`, `\`) and `--` (incidentally via `"valve'; DROP TABLE--"`), but there is no test that inputs `&`, `|`, `!`, `(`, or `)` and asserts they are removed.

```typescript
.replace(/[&|!()]/g, "")
.replace(/--/g, "")
.replace(/\/\*/g, "")
.replace(/\*\//g, "")
```

A future refactor consolidating these regex replacements could inadvertently drop one character class without a test failing. A single additional test case would close this gap.

## Global inflight map — deferred with mitigation path

The module-level `globalInflightMap` cannot be cleanly lifecycle-managed without introducing a factory pattern or service container — both of which change the API surface beyond this PR's scope. The `.finally()` cleanup handles the normal case; hung promises are a pathological scenario that would require composing `requestTimeout.ts` (already available in Phase 9) with the deduplication layer. This composition is a natural follow-up.
